Your learning site should use Tin Can API

There are some great, free online learning sites out there. Here's a few examples:

  • http://duolingo.com/ - learn English, Spanish or German
  • http://www.codecademy.com/ - learn to code
  • http://www.khanacademy.org/ - learn "almost anything" from Algebra to Venture Capital

These free sites are awesome and there are many more out there. They provide learners with outstanding learning, which is mapped out in a clear path and which can be tracked and shared with peers and educators.

These sites are amazing, ground breaking, but they could be better.

Currently if I want to learn Spanish, JavaScript and Venture Capital I'm going to need to register on all three of the above sites. I'll have three separate learning paths and plans and my learning will be tracked across 3 separate systems. Duo the Owl will mail me if I fall behind on my Spanish, but he won't have a clue how my JavaScript is coming along.

By implementing the Tin Can API, each of these sites could allow their users to register just once and complete learning across all three sites and from external providers. Educators and learners themselves could construct a learning plan pulling learning activities from a range of providers. This could all be tracked in one place, so Duo the owl would know about my JavaScript. The tracking data could be shared, so a learner might choose to register with Duolingo as their portal and be tracked on Code Academy by an educator. Learning activities could be linked to and tracked from any system, so learners and educators could use existing Learner management systems which they are already familiar with.

Learners are still visiting the learning provider's site so the learning provider retains control of the learning experience, they just pass tracking data on to tracking tools and sites authorized by the user.

I believe this unlocks huge potential for learning sites and is an important step in producing a complete online education package which pulls the best learning activities from a range of providers. If you work for or are involved with one of the sites listed above, or a similar site, I urge you to look in more detail at the Tin can API and to start to dream about the possibilities it could open up.
